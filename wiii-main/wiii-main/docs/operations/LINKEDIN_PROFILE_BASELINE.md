# LinkedIn Profile Baseline (Wiii owner)

Status: Active
Owner: Project leadership
Last updated: 2026-06-01

Goal: make the profile credible and aligned with Wiii’s positioning **before** publishing the first public post.

## Must-have (blockers before first post)

- **Photo**: clear headshot, neutral background, consistent with professional identity.
- **Headline**: specific and scannable (avoid vague “AI enthusiast”).
- **About**: 6–10 lines, concrete domains + what you’re building + constraints (production-minded, safety, evidence).
- **Featured**: at least 1 item that proves the work (public repo link, architecture doc, demo video, or a clean one-page writeup).
- **Experience / Project**: Wiii described as a product/engineering effort with responsibilities + outcomes (no inflated metrics).
- **Public URL**: clean `linkedin.com/in/<handle>` if available.

## Recommended (do next)

- **Banner**: simple Wiii visual identity (no dense text).
- **Skills**: 15–25 skills aligned to actual work (FastAPI, Postgres/pgvector, RAG, Tooling/MCP, Tauri, React, TypeScript).
- **Projects**: “Wiii” project entry with 2–3 verified highlights.
- **Open-source credibility**: pin repo/docs that show engineering depth.
- **Contact**: optional email for serious inquiries (no personal phone).

## Quality checklist (quick)

- No buzzword stuffing; every claim should be explainable in one sentence.
- Consistent naming: “Wiii” and “agentic RAG platform” used the same way everywhere.
- Audience clarity: product + engineering + education technology.
- Safety posture: mention “privacy/tenant isolation/tool boundaries” only if implemented and verified.
